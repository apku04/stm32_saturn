## Root Cause

**The bug was never about which I2C peripheral (I2C2 is correct).**  
**The bug was not even about pins (PB8/PB9 is correct).**  

**The bug was: using AF4 instead of AF6**

On STM32U073CBT6, PB8/PB9 can be routed to either:
- AF4 → I2C1 peripheral (original code, wrong)
- AF6 → I2C2 peripheral (correct, 1-line fix)

Previous "fixes" (moving to PB6/7, then PB13/14) were chasing symptoms instead of 
addressing the root cause. Eisenhorn's independent PCB verification prevented a 
third wrong migration.

### Beacon Payload (V2)
```
[0:1]   shunt_mv (int16, big-endian)  — INA219 reg 0x01
[2:3]   bus_mv (uint16, big-endian)   — INA219 reg 0x02 >> 3
[4:5]   bat_mv (uint16, LE)           — ADC result × 6600/4096
[6]     chg (uint8)                   — charge status
```

### Monitor Parser
- BEACON_PATTERN_V2: `[BEACON] shunt=... bus=... bat=... chg=... entries=...`
- Legacy fallback: `[BEACON] bat=... sol=... chg=...` (5-byte format)
